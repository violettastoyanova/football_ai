# Football Manager

## Описание

Проектът представлява конзолно приложение на Python за управление на футболни клубове и футболисти.

Системата използва SQLite база данни, която се създава автоматично чрез SQL файл.  
Реализирани са релации между таблиците чрез външни ключове (FK).

---

## Използвани технологии

- Python 3
- SQLite
- SQL (DDL и DML)
- Регулярни изрази (Regex) за обработка на чат команди

---

## Структура на проекта

- src/ – съдържа Python кода
- sql/ – съдържа файла schema.sql
- docs/ – документация и ER диаграма
- README.md – описание и инструкции

---

## База данни

### Таблици:

### clubs
- club_id (PK)
- name
- city
- founded_year

### players
- player_id (PK)
- full_name
- birth_date
- nationality
- position
- number
- status
- club_id (FK → clubs)

Връзка:  
Един клуб може да има много играчи (1:N).

---

## Инициализация на базата данни

Базата данни се създава автоматично при първо стартиране на програмата.

Файлът `schema.sql`, който се намира в папка `sql/`, съдържа:

- създаване на таблиците (clubs, players)
- дефиниране на първични и външни ключове
- начални (seed) данни за клубове и играчи

При стартиране на приложението Python изпълнява `schema.sql` и създава файл `football.db`.

Ако желаете да създадете базата отново:

1. Изтрийте файла `football.db`
2. Стартирайте отново `main.py`

## Функционалности

### Клубове (CRUD)

- Добавяне на нов клуб
- Извеждане на всички клубове
- Изтриване на клуб (с проверка за налични играчи)

### Футболисти (CRUD)

- Добавяне на нов играч към съществуващ клуб
- Извеждане на играчи по клуб (филтър)
- Промяна на номер
- Промяна на позиция
- Промяна на статус (активен/контузен)
- Изтриване на играч

---

## Валидации

- Позиция ∈ {Вратар, Защитник, Полузащитник, Нападател}
- Номер ∈ [1–99]
- Валиден формат на дата: DD-MM-YYYY
- Забрана за дублиране на номер в един и същи клуб
- Забрана за дублиране на играч в един и същи клуб
- Забрана за изтриване на клуб с налични играчи
