# Football Manager

## Описание

Проектът представлява конзолно приложение на Python за управление на футболни клубове, играчи и трансфери.

Системата използва SQLite база данни, която се създава автоматично чрез SQL файл.
Реализирани са релации между таблиците чрез външни ключове (FK).

---

## Използвани технологии

* Python 3
* SQLite
* SQL (DDL и DML)
* Регулярни изрази (Regex) за обработка на чат команди

## База данни

### Таблици:

### clubs

* club_id (PK)
* name
* city
* founded_year

### players

* player_id (PK)
* full_name
* birth_date
* nationality
* position
* number
* status
* club_id (FK → clubs)

### transfers

* id (PK)
* player_id (FK → players)
* from_club_id (FK → clubs)
* to_club_id (FK → clubs)
* transfer_date
* fee (по избор)

---

### Връзки:

* Един клуб има много играчи (1:N)
* Един играч има много трансфери (1:N)

---

## Инициализация на базата данни

Базата данни се създава автоматично при първо стартиране.

Файлът schema.sql съдържа:

* създаване на таблиците
* връзки (FK)
* начални (seed) данни

Ако желаете да създадете базата отново:

1. Изтрийте файла football.db
2. Стартирайте отново main.py

## Функционалности

### Клубове (CRUD)

* Добавяне на клуб
* Извеждане на всички клубове
* Изтриване на клуб (с проверка за налични играчи)

---

### Футболисти (CRUD)

* Добавяне на играч
* Извеждане на играчи по клуб
* Извеждане на всички играчи
* Промяна на номер
* Промяна на позиция
* Промяна на статус
* Изтриване на играч

---

### Трансфери

* Трансфер на играч между клубове
* История на трансфери по играч
* История на трансфери по клуб


---

## Бизнес логика

* Играч може да бъде трансфериран само от текущия си клуб
* Забрана за трансфер към същия клуб
* При трансфер:

  * запис в таблица transfers
  * обновяване на players.club_id
* Операцията е атомична (transaction)

---

## Валидации

* Позиция ∈ {Вратар, Защитник, Полузащитник, Нападател}
* Номер ∈ [1–99]
* Валиден формат на дата: DD-MM-YYYY
* Забрана за дублиране на номер в клуб
* Забрана за дублиране на играч в клуб
* Забрана за изтриване на клуб с играчи
* Забрана за трансфер от грешен клуб

---

## Примерни сценарии

* Успешен трансфер
* Трансфер от грешен клуб (отказ)
* Трансфер към несъществуващ клуб (отказ)
* Трансфер към същия клуб (отказ)
* Показване на история на трансфери
